DRY, what does it mean? Exept for the fact that it doesn't rain ... It is an acronym for Don't Repeat Yourself. Even though this sounds very logical it does happen easily when youre code is getting more and more extended and complex. It happens a lot that at some point you have created a piece of code twice for the same purpose. It is not hard to imagine that this complicates your code and an important part of clean code is to avoid this.

KISS, stands for Keep It Simple, Stupid. It means that you have to look for the most simple way of coding your method. There are often many ways of reaching the same goal and KISS is simply a way for you to choose the most simple way so your code maintains its readability.

Separation of concerns, means that you divide your content in seprate pieces. A good example would be that instead of styling inside the html file, you create a separate CSS file. That way, you keep it clean and structured.

In Summary, clean code means that you keep your code organized and as simple as possible. In the end, who doesn't like a clean and tidy room :)
